Field Explanations
Seed IPs
The starting points for SNMP discovery. The engine contacts each seed IP, collects device information, then follows CDP/LLDP neighbour links outward up to Max Discovery Depth hops. Add at least one core switch or router.
Max Discovery Depth
How many neighbour-link hops to follow from each seed. Depth 0 = seed devices only. Depth 1 = seeds + their immediate neighbours. Higher values cover more of the network but increase runtime proportionally.
Max Workers
Number of devices polled concurrently. Increase this to speed up large syncs. Keep it below your switch SNMP rate-limit threshold — most access switches handle 10–20 concurrent requests without issue.
Cron Expression
Reference value only — NetBox does not execute cron jobs. Copy this into a server cron entry or systemd timer to schedule automatic runs.
External Scheduler Setup

NetBox does not run background jobs directly. Use cron or a systemd timer to execute the sync management command.
