How Credentials Work

When the sync engine contacts a device, it tries each credential in priority order (lowest number first). The first credential that returns a valid SNMP response is used for the rest of that device's data collection.

If no credential succeeds, the device is skipped and an error is recorded in the sync log.

SNMPv2c vs SNMPv3
v2c Community string only. Simpler to configure, but credentials are sent in plain text. Suitable for isolated management networks.
v3 Username + optional authentication (MD5/SHA) and privacy (DES/AES) encryption. Recommended for production environments.
Priority Tips

Assign lower priority numbers to credentials that apply to most of your network. Reserve higher priority numbers for credentials used by a small subset of devices.

If two credentials share the same priority, they are tried in alphabetical order by name.

Unused or retired credentials should be deleted rather than left with a high priority value, as each failed attempt adds latency to every sync.
