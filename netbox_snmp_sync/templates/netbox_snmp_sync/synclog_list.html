What are Sync Logs?

Each time the SNMP sync engine runs, a log entry is created here to record the outcome. Logs capture the start and end time, how many devices were discovered, and whether any errors were encountered.

Sync runs are triggered manually from the command line:

python manage.py sync_snmp
Status Values
Success All devices synced with no errors.
Failed One or more devices raised an error, or the run was terminated by an unhandled exception.
Running Sync started but never recorded a completion — the process was likely interrupted or killed.
Pending Queued but never started.
Scheduling Syncs

The sync engine does not have a built-in scheduler. To run syncs automatically, add a cron job or systemd timer on the NetBox server:

0 * * * * netbox \
  python /opt/netbox/netbox/manage.py \
  sync_snmp

Use --dry-run to test without writing changes, or --seed IP to override the seed IPs from the Schedule page.
